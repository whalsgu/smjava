[JSP]

(JavaServer Pages)
자바 서버 페이지(JavaServer Pages)는 HTML내에 자바 코드를 삽입하여 
웹 서버에서 동적으로 웹 페이지를 생성하여 웹 브라우저에 돌려주는 서버 사이드 스크립트 언어이다. 
Java EE 스펙 중 일부로 웹 애플리케이션 서버에서 동작한다.

자바 서버 페이지는 실행시에는 자바 서블릿으로 변환된 후 실행되므로 서블릿과 거의 유사하다고 볼 수 있다. 
하지만, 서블릿과는 달리 HTML 표준에 따라 작성되므로 웹 디자인하기에 편리하다. 
1999년 썬 마이크로시스템즈에 의해 배포되었으며 이와 비슷한 구조로 PHP, ASP, ASP.NET 등이 있다.

아파치 스트럿츠나 자카르타 프로젝트의 JSTL 등의 JSP 태그 라이브러리를 사용하는 경우에는 
자바 코딩없이 태그만으로 간략히 기술이 가능하므로 생산성을 높일 수 있다.

(배경)
1. 서블릿에서 비즈니스 로직 기능과 화면 기능 분리
2. 디자인은 HTML
3. 비즈니스 로직 JAVA
4. MVC(Model, View, Control)

(구성요소)
1. HTML, CSS, JavaScript
2. JSP 기본 태그
3. JSP 액션 태크
4. 커스텀(Custom) 태크

(처리흐름)
1. JSP 코드를 JSP 컨테이너(tomcat)가 해석을 해서 웹 브라우저에 전송
3. JSP 컨테이너의 처리 단계
   - 변환 단계 : JSP 파일을 자바 파일로 변환(서블릿 코드)
   - 컴파일 단계 : 자바 파일을 클래스 파일로 컴파일
   - 실행 단계 : 자바 클래스 파일을 실행하여 웹브라우저로 전송

(디렉티브 태그)
1. 페이지 디렉티브 태그

3. 태그라이브 디렉티브 태그
   - 개발자가 제공하는 라이브러리 
   
(페이지 디렉티브 태그)
사용법: <%@ page 속성="값" ... %>
1. import: 자바 패키지 임포트
2. session: HttpSession 객체의 사용 여부
3. buffer:  페이지 출력시 사용할 버퍼의 크기
4. autoFlush: 페이지 내용이 출력되기 전 버퍼가 다 채워질 경우 동작을 지정(true)
5. isErrorPage: 페이지 처리 도중 예외가 발생할 경우 예외 처리 담당 페이지 지정 유무(true/false)
6. errorPage: 페이지 처리 도중 예외가 발생할 경우 예외 처리 담당 페이지
7. isELIgnored: EL 사용유무

(인클루드 디렉티브 태그)
사용법: <%@ include file="파일.jsp" %>
1. 다른 JSP 파일을 포함
2. JSP 페이지 코드를 간결
3. JSP 페이지를 재사용
