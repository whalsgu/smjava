[Cookies & Sessions]

(쿠키)
1. 로컬 클라이언트의 영역에 저장되는 파일
2. 웹브라우저가 임시로 사용하는 데이터를 저장
   - 서버에서 수신한 자료를 저장한 후 빠르게 처리를 하기 위해서
   - 사용자 정보, 로그인
   - 검색 정보
   - 히스토리
   - 임시파일
   
3. 쿠키 흐름 
   - 서버에서 쿠키를 생성해서 웹브라우저에 전송하면 웹브라우저는 로컬 저장소에 보관한다.
   - 웹브라우저는 서버에 요청을 보낼 때 쿠키를 해더에 실어서 보낸다.
   - 유효시간이 설정되어 있으면 쿠키는 유효 시간이 지나면 로컬 저장소에 있는 쿠키를 지운다.
    
      
4. 세션 쿠키
   - 웹 브라우저가 사용하는 메모리에 저장
   - 웹 브라우저가 종료되면 쿠키도 소멸
   - 처리흐름
     1) 브라우저가 서버에 접속하여 요청을 보내면 서버는 세션쿠키를 생성해서 브라우저에 전송
     2) 브라우저는 서버가 전송한 쿠키를 받아서 메모리에 보관
     3) 브라우저가 다시 서버에 요청할 때 메모리에 보관된 쿠키를 서버에 전송
     4) 서버는 브라우저가 전송한 쿠키를 보고 어떤 브라우저가 보낸 쿠키인지 식별
     
5. 세션           
   - 웹 브라우저가 서버 사이에 논리적인 연결을 관리
   - 세션 쿠키와 같이 세션 정보를 통해서 연결을 관리
   - 세션 관리를 위한 고유의 키를 발행해서 처리(세션을 관리하기 위한 쿠키와 같은 키 정보)
   - 웹 브라우저가 종료되면 세션도 소멸
   - 처리흐름
     1) 브라우저가 서버에 접속하여 요청을 보내면 서버는 세션키를 생성해서 브라우저에 전송
     2) 브라우저는 서버가 전송한 세션키를 받아서 메모리에 보관
     3) 브라우저가 다시 서버에 요청할 때 메모리에 보관된 세션키를 서버에 전송
     4) 서버는 브라우저가 전송한 세션키를 보고 어떤 브라우저가 보낸 세션키인지 식별
