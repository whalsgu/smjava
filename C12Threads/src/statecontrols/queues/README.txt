[문제]
1. 메시지 큐와 스레드를 이용해서 작업을 처리
2. 메시지 큐의 크기는 10개로 제한
3. 메시지 제공자가 메시지 큐에 데이터를 넣으면
   메시지 소비자는 메시지 큐에서 데이터를 꺼내서 처리한다.
4. 메시지 큐에 데이터 10개가 차면 제공자는 10개 미만이 될 때까지 기다렸다가
   메시지 큐가 10개 미만이 되면 다시 데이터를 넣는다.
5. 소비자는 메시지 큐에 데이터가 있으면 읽는다.
   메시지 큐에 데이터가 없으면 제공자에게 메시지를 달라고 요청한다.   
6. 메시지 큐에 명령어가 "종료" 이면 작업을 종료한다.
