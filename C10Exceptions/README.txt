[10장] 예외 처리

1. 컴파일 오류
   - 컴파일 할 때 문법(Syntax) 오류를 검사(문법 오류)
   - 실행을 시시켠 오류가 발생되면서 실행을 멈춤
   
2. 실행 오류
   - 런타임 오류(Runtime)
   - 논리적인 버그(Bug)
   - 비정상 종료
   
3. 예외처리
   - 실행시간에 비정상인 오류를 만나도 프로그램을 종료하지 않고 처리를 계속한다.
   - 예를 들면 잘못된 데이터가 있다고 해서 종료하기 보다는 
     데이터가 잘못되었다고 알려주고 다시 입력을 받아 처리하도록 유도할 수 있다.
   - 실행오류가 발생되면 사용자에게 오류를 인지할 수 있도록 통보 처리를 위해서 사용.  
   - 자바의 예외 처리 최상위 클래스 : Exception
   - 자바의 실행 예외 처리 클래스 : RuntimeExcpetion
   - RuntimeExcpetion는 Exception을 상속받은 하위 클래스
